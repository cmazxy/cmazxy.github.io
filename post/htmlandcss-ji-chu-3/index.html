1.你对标签语义化的理解是什么？

  1. 标签语义化的可读性更高，便于团队开发维护
  2. 在没有css的情况下，页面也可以呈现出也很好的内容结构和代码结构
  3. 更好的SEO，搜索引擎能更好的理解页面中各部分的关系，可更快更准确的搜索到信息

2.常见的浏览器内核都有哪些？

内核主要分为渲染引擎和 JS 引擎。前者负责页面的渲染，后者负责执行解析 JavaScript。
之后，由于 JS 引擎越来越独立，现在所说的浏览器内核大都指渲染引擎。
目前主流的内核有以下 4 个：
Trident: 由微软开发，即我们熟知的 IE 内核
Gecko: 使用 C++ 开发的渲染引擎，包括了 SpiderMonkey 即我们熟悉的 FireFox
Presto: Opera 使用的内核
Webkit: 前端使用最多的 Chrome 和 Safari 使用的内核

3.说说你对css盒子模型的理解

  1. 基本概念：标准模型+IE模型
  2. css如何设置这两种模式
    2.1
    标准盒模型：box-sizing:content-box;
    标准盒模型的宽度 = margin + border + padding + width(content)
    2.2
    IE盒模型：box-sizing:border-box;
    IE盒模型的宽度 = margin + width (content+padding+border)

两种盒模型的宽度实际上都包括 margin、border 、padding、content这四部分
只是它们对width这个属性的解释不同
ie盒模型对width的解释是 width = content + padding +border
标准盒模型对width的解释是 width = content

4.js放在html的和有什么区别？

js 放在 <head> 中，如果不添加 async属性或defer，当浏览器遇到 script时，会阻塞 DOM 树的构建，进而影响页面的加载。当 js 文件较多时，页面白屏的时间也会变长。
在这个过程中，如果解析器遇到了一个脚本(script)，它就会停下来，并且执行这个脚本，然后才会继续解析 HTML。如果遇到了一个引用外部资源的脚本(script)，它就必须停下来等待这个脚本资源的下载，而这个行为会导致一个或者多个的网络往返，并且会延迟页面的首次渲染时间。
把 js 放到 <body> 里（一般在 </body> 的上面）时，由于 DOM 时顺序解析的，因此 js 不会阻塞 DOM 的解析。对于必须要在 DOM 解析前就要加载的 js，我们需要放在 <head> 中。

5.请说说<script>、<script async>和<script defer>的区别

  1. <script>立即加载并执行相应的脚本。会阻塞后面的文档加载；
  2. <script async>异步加载，脚本执行和后续文档代码加载同时进行；
  3. <script defer>异步加载，脚本的执行需等倒所有文档加载完才执行。
